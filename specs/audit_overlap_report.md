### Two Divergent Mock Data Sets

- SEVERITY: CRITICAL
- FILES: app.py, blueprints/audit/routes.py
- FINDING: The app has two separate mock data functions:
  1. app.py: get_mock_dashboard_data() with 10 categories using keys that
     partially match JSON directories (ai_geo, crawl, eeat, content,
     media, mobile, links, international, local, shopping)
  2. blueprints/audit/routes.py: get_mock_dashboard_data() with 12
     categories using keys that do NOT match JSON directories
     (crawlability_indexation, site_architecture, on_page_seo, etc.)
  The root template (templates/dashboard.html) uses app.py's version.
  The blueprint template (audit/dashboard.html) uses routes.py's version.
  These are completely different category sets with different keys.
- FIX: Choose one source of truth. The app.py keys (crawl, content, links,
  etc.) more closely match the JSON directory structure. Remove the
  duplicate mock data from whichever file is not the primary and ensure a
  single dashboard route serves the canonical data.

### Hyphen vs Underscore Convention

- SEVERITY: INFO
- FILES: json/ directories, all JSON files
- FINDING: JSON directories use hyphens (ai-geo, rich-results) while
  JSON category field values use underscores (ai_geo, rich_results).
  This is a minor but consistent convention difference. The underscore
  form is used in code (Python keys, Jinja variables), the hyphen form
  in filesystem paths.
- FIX: Document this convention explicitly. No code change needed as long
  as a mapping function converts between the two forms.
